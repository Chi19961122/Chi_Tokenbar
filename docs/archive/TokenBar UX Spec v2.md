# TokenBar — UX 行為規格 v2 (handoff spec)

> 本文件定義**行為、內容、狀態、切換規則**，並記錄 v2 已拍板的**視覺方向**。
> 平台：Windows 11。形態：貼邊常駐 widget + 系統匣(tray)圖示。
> v1 → v2 變更：見文末「附錄 A · 決策紀錄」。

---

## 0. 產品意圖(一句話)

讓使用者**不用動腦就知道自己還能不能繼續跑 AI coding 工具**。
價值在「環境感知(ambient awareness)」——**平常近乎隱形，只在該吼時才吼**。

---

## 1. 三層視圖模型

```
① 精簡列(常駐)      → 一眼:我快被切斷了嗎?
      ↓ 點擊展開
② 面板(所有限制條)   → 掃視:哪個工具、哪條限制吃緊?
      ↓ 點某條限制
③ 細節(拆解/趨勢)    → 分析:token 怎麼花的、趨勢
```

---

## 2. 精簡列(①)— 單行,只顯示「最危險一條」

### 2.1 內容 slot(由左到右)

```
🟠   Codex·週    88%    · 2d
色    是哪條      %      倒數
```

- **色點**:狀態色,永不省略(最後的救命信號)。
- **限制標識**:`工具·限制`(如 `Codex·週`、`Claude·5h`)。
- **%** + **倒數**:utilization 百分比、距重置相對時間。

### 2.2 空間不足時的丟棄順序(右 → 左)

`倒數` → `% 數字` → `限制名` → 只剩 `工具 + 色點`。**色點永不丟。**

### 2.3 音量隨危險度遞增(核心互動原則)

| 狀態 | 精簡列表現 |
|---|---|
| 🟢 **安全** | **近乎隱形**:色點 + %,**不顯示倒數**,低對比(預設透明度 ~0.5,可調) |
| 🟠 **接近** | 補上倒數、狀態色描邊、加陰影浮現存在感 |
| 🔴 **鎖定** | 反白填滿 + 緩慢閃爍 + `LOCKED · 1h20m` |

### 2.4 視覺樣式(v2 決定 — 見 §12)

採 **Bold data** 皮膚：膠囊 pill、`Geist Mono` 數字、狀態色描邊(接近)/填滿(鎖定)。

---

## 3.「最危險一條」選擇演算法 + 防跳動

### 3.1 排名規則

- **主規則**:純比 `utilization %`,最高者上榜。
- **唯一例外**:某條 ≥ 警戒門檻 但**數分鐘內即將鬆開**時降級,避免為即將消失的限制報警。
- 不做「% × 重置時間」加權。

### 3.2 遲滯(hysteresis)— 必做

- **黏性**:目前上榜那條保持顯示,**除非另一條超過它 ≥ 5 個百分點**才換人。
- **最短停留**:換上榜後**至少停留 30–60 秒**才允許再換。

---

## 4. 貼邊列 vs 系統匣圖示(分工不重複)

| 元件 | 顯示內容 |
|---|---|
| **貼邊精簡列** | §2 的完整單行(色 + 名 + % + 倒數) |
| **系統匣圖示** | **燃料膠囊(fuel capsule)**：一條會排空的電量/油量條,填滿比例 = 最危險一條的 utilization,顏色 = 狀態色。收起貼邊列時,光看角落就知道要不要緊 |
| **hover 系統匣** | tooltip **攤開全部限制條**(唯一不受「只顯示一條」限制的出口) |
| **點擊(列或圖示)** | 展開面板 ② |

### 4.1 系統匣圖示規格(v2 決定：3D 燃料膠囊)

- 形狀：橫向圓角膠囊,外框 track 深灰,內部填色條。
- **安全**:深灰外框 + 綠色填色(填至 utilization%)。
- **接近**:深灰外框 + 琥珀填色。
- **鎖定**:**外框轉紅、內部填白、填滿 100%**（最醒目）。
- 縮到 16px 仍需可辨識；顏色是主要訊號,填滿比例是次要。
- 選配：可疊工具字母(C / X)以區分 Claude / Codex。

---

## 5. 面板(②)— 限制卡片

### 5.1 頂部 tab

`All · Claude · Codex`(多工具切換,可擴充)。

### 5.2 單張限制卡解剖(v2 樣式：2A Tinted bar cards)

```
┌─────────────────────────────────────────┐
│ 5h Session                        🟢 42% │  名稱(左) + 大等寬 % 數字(右)
│ ▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░                   │  粗進度條(主視覺)
│ 620K / 1.2M tokens                        │  絕對值(有才顯示)
│ Resets by ~3:40 PM · in 2h 13m            │  浮動重置
└─────────────────────────────────────────┘
```

- **卡片背景依狀態上色**：安全=中性、接近=琥珀底、鎖定=紅底(含 `LOCKED` 標籤)。
- **% 用大號 `Geist Mono`**（右上,狀態色）。
- **絕對值(已用/上限)是選配**:Claude 訂閱基本**只有 %**(官方 2026 起不公布絕對數字),**版面不得為它預留固定位置**。僅 Codex(token 數)與 extra credit 有實數。
- **重置文案要誠實**:rolling window 是浮動的,用 `Resets by ~` + 相對倒數,**不可寫死固定時鐘**。

## 6. 狀態機(所有卡片 / 精簡列都須涵蓋)

| 狀態 | 何時 | 精簡列表現 | 卡片表現 |
|---|---|---|---|
| **正常** | 有即時數據 | `🟢 42%`(近乎隱形) | 中性卡 + 綠色 |
| **接近上限** | 過門檻(75 / 90%) | `🟠 88% · 2d` + 通知 | 琥珀底卡 + 琥珀 |
| **已鎖定** | 撞牆 | `🔴 LOCKED · 1h20m` | 紅底卡 + `LOCKED` 標籤 |
| **資料過期** | 快取超時 / 無新活動 | 灰階 + `~` 或「5m ago」 | 灰階 + `~%` |
| **來源失效** | Claude 非官方 endpoint 掛掉 | **退回本機 token 估算 + 標「估算」**,不得空白 | 虛線邊 + 「估算」徽章 |
| **工具未在跑** | Codex/Claude 沒開 | 淡出該工具,**不顯示成 0%** | 淡出,不顯示成 0% |

> 「來源失效」是本產品特有風險:Claude 即時限制 % 押在未公開 API。此態的降級體驗直接決定產品可信度。

## 7. 限制清單(資料來源真相)

### Claude Code（4 條）

| 限制 | 資料來源 | 有絕對值? |
|---|---|---|
| 5h Session | `GET /api/oauth/usage` → `five_hour.utilization` + `resets_at` | 否(僅 %) |
| 週(全模型) | `seven_day.utilization` + `resets_at` | 否 |
| 週(Opus) | `seven_day_opus` | 否 |
| Extra credit(付費備援池) | `extra_usage`(is_enabled / monthly_limit / used_credits) | **是** |

> 現況佐證(2026)：官方 Settings > Usage 同時顯示 5h session 與週限制進度條,且週限制**分別顯示 Opus only 與所有其他模型**的重置時間。兩者皆為**滾動視窗**(5h 從首個請求起算;週為滾動 7 天,非週一歸零)。額度在 Pro/Max 上**跨 Claude Code / 網頁 chat / Desktop 共用**。官方不再公布絕對 token/則數,只給相對容量(Pro 基準、Max 5x、Max 20x)。
> ⚠️ 前三條全靠**未公開 endpoint**,需 OAuth token + beta header,可能無預警失效 → 見狀態機「來源失效」。

### Codex（3 條）

| 限制 | 資料來源 | 有絕對值? |
|---|---|---|
| 5h 視窗 | 本機 `~/.codex/sessions/**/rollout-*.jsonl` 的 `rate_limits` 快照 | 是(token) |
| 週視窗 | 同上 | 是 |
| Credit 餘額 | ❓ 本機檔通常沒有,需 `platform.openai.com`,難自動化 → 顯示為「connect」態 | — |

> 現況佐證(2026)：Codex 以 5h 視窗 + 週配額計；本機 messages 與 cloud tasks 共用同一 5h 池；已改為 token/credit 計價,額度用完可買 credit。Codex CLI 內 `/status` 可查,但**無公開即時 API** → 這正是 TokenBar 的切入點。
> Codex 的限制 % **寫在本機檔**,不需未公開 API → 最穩,建議當 MVP 起點。

## 8. 通知規則

- 觸發:門檻(75% / 90%,可設)各一次 + 鎖定時一次。
- 通道:Windows 原生 toast。
- **抑制**:同一條限制觸發後**30 分鐘內不重複吵**。
- 文案可帶省量建議(Codex:切 mini;Claude:/compact、換 Sonnet)。

---

## 9. 細節層(③)內容(資料皆來自本機 JSONL,穩定無風險)

- **用量拆解**:今日 input / output / cached(+ Codex reasoning)token;各工具佔比、各模型佔比。
- **趨勢**:今日 / 本週切換,逐日長條;「本週 session 數」由 5h 視窗推算。
- **成本估算(選配)**:Codex 走 credit;Claude 訂閱制標「已含於方案」。為估算值,非官方帳單。

---

## 10. 常駐體驗(Windows)

貼邊(上/下緣可選)、置頂、開機自動啟動、可收合成單行。點某條限制展開細節。
> 「選單列(menu-bar)」是 macOS 概念,Windows 對應為**系統匣(tray)+ 貼邊 widget**。

---

## 11. 不可被視覺推翻的行為約束

1. 精簡列只顯示最危險一條,安全態近乎隱形。
2. 色點永不省略。
3. 絕對值版面為選配,Claude 卡片不得因缺絕對值而破版。
4. 重置時間必須呈現為浮動(`~` + 相對倒數)。
5. 「來源失效」必須有降級樣式,不得空白。
6. 遲滯(±5% / 停留 30–60s)必須實作,避免精簡列狂閃。

---

## 12. 視覺方向(v2 已拍板)

原 v1「留給視覺的開放項」已決定如下：

| 項目 | 決定 |
|---|---|
| **整體皮膚** | **Bold data**：深色為主、大號 `Geist Mono` 數字、高對比、狀態色底卡片、粗進度條 |
| **面板卡片版面** | **2A Tinted bar cards**：狀態色底 + 單一大數字 + 粗 meter(平穩好掃) |
| **系統匣圖示** | **3D 燃料膠囊**：會排空的電量條,顏色=狀態(見 §4.1) |
| **狀態三色** | 安全 `#34d399` / 接近 `#fbbf24` / 鎖定 `#f87171`;過期・來源失效 `#8a929d` |
| **主色(品牌/安全)** | 綠 `#34d399`(可調) |
| **字體** | `Geist`(介面) + `Geist Mono`(數字/倒數/token) |
| **安全態透明度** | 預設 ~0.5,可調 |
| **深/淺色** | 兩者皆支援,含切換 |

## 附錄 A · 決策紀錄(v1 → v2)

- **限制模型修正**：移除原「月方案」為獨立限制的誤解;Claude 訂閱只有 5h + 週(全模型)+ 週(Opus),第四欄改為 `extra_usage` 月度 credit 池。
- **視覺方向定案**：Bold data 皮膚 + 2A 面板 + 3D 燃料膠囊系統匣圖示(見 §12)。
- **平台確認**：Windows 11(tray + 貼邊 widget),非 macOS menu-bar。
- **互動原型**：三層導覽(精簡列→面板→細節)+ 分頁篩選 + usage 情境切換,已於 `TokenBar Prototype.dc.html` 實作。
