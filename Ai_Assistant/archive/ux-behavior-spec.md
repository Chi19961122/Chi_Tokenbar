# TokenBar — UX 行為規格(handoff spec)

> 本文件只定義**行為、內容、狀態、切換規則**——一切**皮膚無關**。
> 視覺調性(配色、字體、圓角、質感)交由 Claude Design 決定,見文末「留給視覺的開放項」。
> 平台:Windows 11。形態:貼邊常駐 widget + 系統匣圖示。

---

## 0. 產品意圖(一句話)

讓使用者**不用動腦就知道自己還能不能繼續跑 AI coding 工具**。
價值在「環境感知(ambient awareness)」,不在功能多寡——所以**平常近乎隱形,只在該吼時才吼**。

---

## 1. 三層視圖模型

```
① 精簡列(常駐)      → 一眼:我快被切斷了嗎?
      ↓ 點擊展開
② 面板(所有限制條)   → 掃視:哪個工具、哪條限制吃緊?
      ↓ 點某條限制
③ 細節(拆解/趨勢)    → 分析:token 怎麼花的、趨勢
```

---

## 2. 精簡列(①)— 超精簡單行,只顯示「最危險一條」

### 2.1 內容 slot(由左到右)

```
🟠   Codex·週    88%    · 2d
色    是哪條      %      倒數
```

- **色點**:狀態色,永不省略(最後的救命信號)。
- **限制標識**:`工具·限制`(如 `Codex·週`、`Claude·5h`)。
- **%**:utilization 百分比。
- **倒數**:距重置的相對時間(`2d`、`2h13m`)。

### 2.2 空間不足時的丟棄順序(右 → 左)

`倒數` → `% 數字` → `限制名` → 只剩 `工具 + 色點`。
**色點永不丟。**

### 2.3 音量隨危險度遞增(核心互動原則)

| 狀態 | 精簡列表現 |
|---|---|
| 🟢 **安全** | **近乎隱形**:只有色點 + %,**不顯示倒數**,低對比、不搶眼 |
| 🟠 **接近** | 補上倒數、顏色轉明顯 |
| 🔴 **鎖定** | 整條反白/最醒目 + `LOCKED · 1h20m` |

> 已確認:安全狀態走「近乎隱形」。平常近乎看不見,只在接近/鎖定時才浮現存在感。

---

## 3.「最危險一條」選擇演算法 + 防跳動

### 3.1 排名規則

- **主規則**:純比 `utilization %`,**最高者上榜**。
- **唯一例外**:某條 ≥ 警戒門檻 但**數分鐘內即將鬆開**時,將其降級,避免為一條即將消失的限制報警。
- 不做「% × 重置時間」加權(rolling window 重置非乾淨歸零,加權難解釋也難實作)。

### 3.2 遲滯(hysteresis)— 必做,否則會在相近值間狂閃

- **黏性**:目前上榜那條保持顯示,**除非另一條超過它 ≥ 5 個百分點**才換人。
- **最短停留**:換上榜後**至少停留 30–60 秒**才允許再換。

> 已確認:採預設 ±5% / 停留 30–60s。

---

## 4. 貼邊列 vs 系統匣圖示(分工不重複)

| 元件 | 顯示內容 |
|---|---|
| **貼邊精簡列** | §2 的完整單行(色 + 名 + % + 倒數) |
| **系統匣圖示** | 更極端:**圖示底色 + badge 數字(%)**。收起貼邊列時,光看角落顏色就知道要不要緊 |
| **hover 系統匣** | tooltip **攤開全部限制條**(唯一不受「只顯示一條」限制的出口) |
| **點擊(列或圖示)** | 展開面板 ② |

---

## 5. 面板(②)— 限制卡片

### 5.1 頂部 tab

`All · Claude · Codex`(對應多工具切換)。

### 5.2 單張限制卡解剖

```
┌─────────────────────────────────────────┐
│ 5h Session              🟢 42%           │  名稱 + 狀態色 + %
│ ▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░                  │  進度條(主視覺)
│ Resets by ~3:40 PM (in 2h 13m)           │  浮動重置
└─────────────────────────────────────────┘  (絕對值有才顯示,沒有就省)
```

- **絕對值(已用/上限)是選配**:Claude 訂閱基本**只有 %**(官方 2026 起不公布絕對數字),**版面不得為它預留固定位置**,否則 Claude 卡片永遠開天窗。僅 Codex(token 數)與 extra credit 有實數。
- **重置文案要誠實**:rolling window 是浮動的,用 `Resets by ~` + 相對倒數,**不可寫死固定時鐘**(避免誤解為每日固定時間重置)。

## 6. 狀態機(所有卡片 / 精簡列都須涵蓋)

| 狀態 | 何時 | 精簡列表現 | 備註 |
|---|---|---|---|
| **正常** | 有即時數據 | `🟢 42%`(近乎隱形) | |
| **接近上限** | 過門檻(75 / 90%) | `🟠 88% · 2d` + 通知 | |
| **已鎖定** | 撞牆 | `🔴 LOCKED · 1h20m` | 最重要的一態 |
| **資料過期** | 快取超時 / 無新活動 | 灰階 + `~` 或「5m ago」小字 | |
| **來源失效** | Claude 非官方 endpoint 掛掉 | **退回本機 token 估算 + 標「估算」**,不得空白 | TokenBar 特有,決定可信度 |
| **工具未在跑** | Codex/Claude 沒開 | 淡出該工具,**不顯示成 0%** 誤導 | |

> 「來源失效」是本產品特有風險:Claude 即時限制 % 押在未公開 API。此態的降級體驗直接決定產品可信度。

---

## 7. 限制清單(資料來源真相)

### Claude Code（4 條)

| 限制 | 資料來源 | 有絕對值? |
|---|---|---|
| 5h Session | `GET /api/oauth/usage` → `five_hour.utilization` + `resets_at` | 否(僅 %) |
| 週(全模型) | `seven_day.utilization` + `resets_at` | 否 |
| 週(Opus) | `seven_day_opus` | 否 |
| Extra credit(付費備援池) | `extra_usage`(is_enabled / monthly_limit / used_credits) | **是** |

> 註:原清單的「月方案」不存在為獨立滾動限制——訂閱只有 5h + 週。此欄改對應 `extra_usage` 的月度 credit 池。
> ⚠️ 前三條全靠**未公開 endpoint**,需 OAuth token + beta header,可能無預警失效 → 見狀態機「來源失效」。

### Codex（3 條)

| 限制 | 資料來源 | 有絕對值? |
|---|---|---|
| 5h 視窗 | 本機 `~/.codex/sessions/**/rollout-*.jsonl` 的 `rate_limits` 快照 | 是(token) |
| 週視窗 | 同上 | 是 |
| Credit 餘額 | ❓ 本機檔通常沒有,需 `platform.openai.com`,難自動化 | — |

> Codex 的限制 % **寫在本機檔**,不需任何未公開 API → 最穩,建議當 MVP 起點。

---

## 8. 通知規則

- 觸發:門檻(75% / 90%,可設)各一次 + 鎖定時一次。
- 通道:Windows 原生 toast。
- **抑制**:同一條限制觸發後**30 分鐘內不重複吵**,否則變騷擾。
- 文案可帶省量建議(Codex:切 mini;Claude:/compact、換 Sonnet)——純文案,無技術門檻。

---

## 9. 細節層(③)內容(資料皆來自本機 JSONL,穩定無風險)

- **用量拆解**:今日 input / output / cached(+ Codex reasoning)token;各工具佔比、各模型佔比。
- **趨勢**:今日 / 本週切換,逐日長條;「本週 session 數」由 5h 視窗推算。
- **成本估算(選配)**:Codex 走 credit;Claude 訂閱制標「已含於方案」。為估算值,非官方帳單。

---

## 10. 常駐體驗(Windows)

貼邊(上/下緣可選)、置頂、開機自動啟動、可收合成單行。點某條限制展開細節。
> 「選單列(menu-bar)」是 macOS 概念,Windows 對應為**系統匣(tray)+ 貼邊 widget**。

---

## 11. 留給視覺的開放項(交給 Claude Design)

以下**刻意不定**,由視覺設計決定:

- 整體調性:極簡數據儀表 vs 活潑卡片。
- 三段狀態色(綠/琥珀/紅)的實際色值與無障礙對比。
- 「近乎隱形」安全態的具體低對比程度、字重、透明度。
- 進度條樣式(實心/區塊/漸層)、字體、圓角、間距。
- 系統匣圖示的圖形與 badge 呈現。
- 「LOCKED」反白態的視覺強度。

**但下列行為約束不可被視覺推翻**:
1. 精簡列只顯示最危險一條,安全態近乎隱形。
2. 色點永不省略。
3. 絕對值版面為選配,Claude 卡片不得因缺絕對值而破版。
4. 重置時間必須呈現為浮動(`~` + 相對倒數)。
5. 「來源失效」必須有降級樣式,不得空白。
