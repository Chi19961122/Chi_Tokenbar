# TokenBar 三樣態優化計畫(執行版)

> 2026-07-16 定稿。來源:使用者提供之規格(以該規格為準)+ 四項拍板決策 + Blind Spot Pass 結論。
> 執行原則:只做本檔列出的項目,不擴 scope;完成一階段再進下一階段;每階段結束跑既有測試 + 驗收清單。
> 視覺概念板:claude.ai/design 專案「TokenBar Design System」→ 群組「戰報 Share x6」。
> 全貌與 C+ 圖表視覺:concept 稿 artifact df66099c-963f-4923-9262-6d249d570198(島嶼品牌 icon [lobe-icons]、Limits 精簡、Usage GitHub 式熱力圖 + 活動類型 donut + 專案維度長條)。

## 0. 產品邊界(不可違反)

1. 島嶼 + Limits = 訂閱還剩多少電(% / 重置)。
2. Usage = 用量統計與圖表。
3. Share 戰報 = 所選區間的累計用量(token + est. $),不是額度電表。
4. 戰報拆分用絕對 token;顯示 % 必須是「佔本區間總量」,禁止與訂閱限額 % 混用。
5. 不做:排行榜、聊天、雲端徽章、書券風、展覽入場票、25+ agent 全量額度。
6. 平台維持 Windows + Tauri 2 + vanilla TS。

## 預設值(Blind Spot Pass 補齊,可再改)

- `quotaNote` 預設 off;僅 `island_card` 風格預設 on(單行、非主視覺)。
- `share_style` 預設 `statement`;`share_style`、`share_range` 都記住上次選擇。
- 匯出尺寸預設橫式 1200×675(16:9);直式列後續。
- 頂列 `Refresh in Ns` 倒數(v0.2.1)保留,spec 未提及即不動。
- month range 若本機 log 不足 30 天:有多少算多少 + 標註起始日。
- OpenCode/Gemini 的 Limits 判準:本機官方檔案有穩定 limit 欄位才做;否則僅 Usage。

## 與現況 v0.2.1 的衝突(spec 優先,實作時移除)

## 執行順序(建議;理由:依賴關係)

規格文件順序為 1 戰報 → 6 i18n,但戰報文案依賴字典、Limits/Island 是小改,建議:

### 階段 A:i18n 架構(§6)
- [ ] `locale` 設定欄位 + 跟隨系統 fallback(zh-TW 系統 → zh-TW,其餘 en)
- [ ] 字典模組(面板/設定/通知/戰報全文案 key 化);先建 en 全量 + zh-TW 全量

- [ ] 島嶼限額短標固定英文,不進字典
- Done:切 zh-TW/en 即時重繪、無關鍵漏翻。

### 階段 B:島嶼 + Limits 瘦身(§2 + §3)
- [ ] settings 新欄位 + `config.rs` 遷移(缺欄填預設不炸):`expand_default`(預設 compact)、`island_pin_claude`、`island_pin_codex`、`island_aux`、`locale`、`reset_display`(預設 `relative`=倒數,另 `clock`=時刻)
- [ ] `island.ts` 顯示矩陣:Near/Locked 顯示英文短標(5h/wk/模型短名),normal 不顯示;Near `{left}% · {reset}`、Locked `0% · {reset}`;`{reset}` 依 `reset_display`:`relative`=倒數(距重置,非 runway)或 `clock`=時刻(絕對時間,跨日補日標 明/週日,跟 locale 24h/12h)
- [ ] 釘選與 ranking:`auto | 5h | week | model:<id>`;釘了無資料 → `—`,不靜默退回 auto
- [ ] aux:`off | tok_per_min | cost_today`
- [ ] 右鍵 context menu(D4):釘選、provider 切換、設定、隱藏
- [ ] 展開路徑走 `expand_default`(預設進 Limits 精簡)
- [ ] `panel.ts` 刪 detail view;relogin 留列表列;刪 pace 文案;Near/Locked 左側色條加粗;Unavailable/Estimate 與真 0% 視覺區隔;reset 時間依 `reset_display` 呈現(與島嶼一致)
- [ ] 刪死碼;更新 `island.test.ts` 與相關測試

### 階段 C:Usage 詳細模式(§4)

- [ ] subtab 收斂:daily 併入 overview;agents 併入 group toggle

### 階段 C+:Usage 進階維度(承 §4;獨立於階段 C 之後發布)
> 由階段 C 的「後續」抽出獨立成階段。理由:三項各自需要**新資料源**(活動分類、專案歸屬、熱力聚合),與 C 的「面板/Usage 瘦身」性質不同;硬併會把 C 從瘦身變成加三個資料源,scope 與工期跳一級。C 先出、本階段再排。
- [ ] 活動熱力圖(2D):GitHub 式日曆貢獻圖(X=月份、Y=星期、每格=每天,深淺=當日 token),範圍跟 month range;先只做 2D。3D(WebGL 柱體,高度=量)視效能與需求再議,預設不做
- [ ] 活動類型:以 donut 佔比圖呈現(coding/edit、search/read、review/verify、test/build…);資料層需先能分類活動,現況 log 未必帶類型欄位 → 先確認來源或補標註,無法分類則不顯示假類別
- [ ] 專案維度:by-project token 水平長條排名,**僅本機 Usage 顯示**;硬限制:專案名一律不進戰報(§0),`buildShareData` 維持禁專案名
- [ ] 圖表沿用階段 C 的「數字 /(佔區間總量)%」標籤慣例與 cache;本機 log 不足即標註起始日
- [ ] 假資料 + 測試(熱力聚合 / 類型分類 / 專案聚合各一單元)
- Done:Usage 可看 2D 熱力圖、活動類型佔比、專案用量;戰報不受影響、仍無專案名。
- 版本:v0.3.x(晚於階段 C 的 minor,可獨立發布)

## 風險與注意

1. html-to-image 匯出需字型就緒:Geist/Noto 以 data URI 或確保 document.fonts.ready 後再 render,避免匯出圖 fallback 字型。
2. i18n 重繪:切 locale 走全量 re-render(現有 render 函式皆冪等,成本低);設定 id 不動。`reset_display=clock` 的絕對時間用固定 locale 格式化(避免 v0.2.1 fmtReset 語系洩漏),12/24h 依 locale。
3. settings 遷移:舊檔缺欄一律填預設;`island_pin_*` 的 `model:<id>` 需驗證 id 仍存在,不存在顯示 `—`。
4. 刪 detail view 前先確認無其他入口引用(tray、快捷)。
5. 版本規劃:階段 A+B 出 v0.3.0,C 出 v0.3.x,C+(Usage 進階維度)出更後的 v0.3.x,D 出 v0.4.0,E 出 v0.5.0(每階段獨立可發布)。
6. 參考檔 `share-concepts/index.html`(grok 資料夾)本機不可及;視覺以 Design 專案概念板為準,若日後取得 mock 再對照。
